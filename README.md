# Grid Engineering

[한국어](README.md) | [English](README_EN.md)

Minecraft 1.21.1 NeoForge용 전력 전송 및 전력망 설계 모드입니다.

현재 단계는 연결형 전선, FE 전송, 암페어 제한, 전선 화재와 전력 제어 장치를
검증하는 초기 프로토타입입니다.

## 현재 구현

- 중앙 2×2픽셀 단면의 6방향 연결형 구리 전선
- 인접한 구리 전선을 자동으로 연결
- 구리 전선을 들고 웅크린 채 기존 전선 면을 우클릭하여 빈 방향으로 끝을 연장/회수
- 전선 커터로 선택한 연결을 끊거나 다시 연결
- 전선 커터를 들고 웅크린 채 가지를 우클릭하여 자동 → 입력 → 출력 포트 순환
- 입력 포트는 밝은 구리색, 출력 포트는 어두운 구리색으로 표시
- 주석 전선은 LV 65,536V·1A, 구리 전선은 MV 524,288V·1A
- 전선 재질: 주석·납·니켈·구리·철·금·일렉트럼·강철·텅스텐·티타늄·NTT 합금·뉴트로늄·NBB 합금
- 모든 재질에 1mm·2mm·4mm·8mm·16mm 굵기를 제공하며 허용 전류는 재질 기본 A × 굵기
- 모든 전선은 공용 `block/cable` 철 블록 기반 텍스처에 재질 색상을 틴트하여 렌더링
- 구리 전선은 0.2%/m/A, 주석 전선은 0.5%/m/A의 경로별 송전 손실 적용
- 여러 장치가 연결되면 Power of Two Choices 방식으로 덜 받은 장치를 골라 전력을 균등 분배
- 허용 전압보다 높은 전압이나 허용 암페어보다 큰 전류가 흐르면 전선 하나가 화재로 소실
- 낮은 전압은 상위 전압 전선으로 정상 전송
- 실제 전류가 1A를 넘으면 연결망의 구리 전선 하나가 전선 화재로 소실
- 네트워크 모니터로 전압, 현재/최대 암페어, 입력·도착·손실 FE/t와 출발·도착 좌표 확인
- 전류 조절기는 정면 한 면으로 전력을 받고 나머지 다섯 면으로 설정한 전압×전류를 균등 출력
- 전류 조절기는 전력을 월드에 저장하지 않는 통과형 장치이며 입력이 끊기면 출력도 즉시 소멸
- 조절기 입력면은 위·아래를 포함한 6방향으로 설치할 수 있고 입력면 앞 장치에서 전력을 자동 추출
- 조절기는 임의의 숫자 전압을 입력할 수 있으며 전선 등급은 다음 상위 티어로 올림 판정
- 테스트 배터리와 전류 조절기는 전용 GUI에서 전압 티어와 1~64A를 설정
- 테스트 배터리도 임의 숫자 전압과 암페어로 출력을 설정
- 네트워크 모니터 아이콘의 Grid Engineering 전용 크리에이티브 탭 제공
- 배터리·조절기·전선망 내부는 long 기반 전용 전력을 사용하고 FE는 외부 모드 경계에서만 변환
- 절단 상태를 월드에 저장하고 클라이언트에 동기화
- 연결된 전선망을 통한 전압×전류 기반 FE 전송
- 입력/무한 출력/버퍼 모드를 제공하는 FE 테스트 배터리
- 개발 실행 환경에 Mekanism, 일반 배포판 EMI, Jade 포함
- 일반 재료 조합법은 제공하지 않으며 전선 굵기 합치기·분해 조합법만 제공

전선, 커터, 네트워크 모니터, 테스트 배터리와 전류 조절기는 크리에이티브 모드의 레드스톤 블록 탭에서 찾을 수 있습니다.
네트워크 모니터로 전선을 우클릭하면 해당 전선망의 실시간 전송 상태가 채팅에 표시됩니다.
전선 커터로 연결된 전선의 가지 부분을 우클릭하면 해당 연결이 토글됩니다.
웅크린 채 우클릭하면 해당 가지의 포트 모드를 설정합니다. 입력은 전선망에서
장치로 넣는 방향이고, 출력은 장치에서 전선망으로 빼오는 방향입니다.

테스트 배터리는 기본적으로 FE 입력 모드입니다.

- 우클릭: 전용 GUI 열기
- GUI에서 입력/무한 출력/버퍼 모드, 전압 티어와 1~64A 설정

전류 조절기는 모델의 정면 한 면이 입력이고 뒤·옆·위·아래 다섯 면이 출력입니다.
GUI에서 설정한 전압×전류를 틱당 최대 출력량으로 사용합니다.

무한 출력 배터리를 LV 1A로 설정하고 입력 배터리에 연결하면 틱당 65,536 FE가 흐릅니다.
Mekanism 장치도 NeoForge FE capability를 노출하는 면에 전선을 붙이면 같은 방식으로
연결됩니다. 현재 전송기는 최종 전력망 엔진 이전의 호환성 검증용 구현입니다.

## 개발 환경

- JDK 21
- Minecraft 1.21.1
- NeoForge 21.1.234
- ModDevGradle

## IntelliJ IDEA에서 열기

1. IntelliJ IDEA에서 이 폴더를 엽니다.
2. Gradle 프로젝트 가져오기를 승인합니다.
3. Gradle JVM을 JDK 21로 지정합니다.
4. 초기 동기화가 끝나면 `runClient` 또는 생성된 Minecraft Client 구성을 실행합니다.

명령줄 빌드:

```powershell
.\gradlew.bat build
```

전체 구현 계획은 [docs/IMPLEMENTATION_PLAN.md](docs/IMPLEMENTATION_PLAN.md)를 참고하십시오.
